## Part 1: Network Hardening Tools and Methods

Three hardening tools the organization should implement to address the vulnerabilities found:

1. **Multi-Factor Authentication (MFA)**
2. **Strong Password Policies**
3. **Firewall Maintenance and Configuration**

---

## Part 2: Explanation

### 1. Multi-Factor Authentication (MFA)
MFA requires users to verify their identity using two or more methods before accessing the system. Some MFA methods include fingerprint scans, ID cards, PIN numbers, and passwords. Even if a password is stolen or shared, the attacker cannot access the system without the second verification factor. MFA also reduces password sharing since the recipient would need additional authentication. This should be implemented immediately and maintained permanently.

### 2. Strong Password Policies
Password policies should include rules regarding password length, acceptable characters, and a prohibition on password sharing. Policies should also include account suspension after a certain number of failed login attempts to prevent brute force attacks. Requiring complex passwords, frequent password updates, and disallowing password reuse will make it significantly harder for malicious actors to gain unauthorized access.

### 3. Firewall Maintenance
Firewall maintenance involves regularly checking and updating security configurations to stay ahead of potential threats. Network administrators should ensure firewall rules reflect the most up-to-date standards for allowed and denied traffic. Suspicious traffic sources should be blocked. Firewall rules should be updated whenever a security event occurs. This measure protects against various DoS and DDoS attacks.
